## 1. HAM³：多模态多智能体系统的层次化攻击框架
- **来源**: arXiv
- **作者**: Hao Zhou et al.
- **链接**: http://arxiv.org/abs/2605.13213v1
- **方向**: 🤖 AI/ML（多智能体安全）
- **TL;DR**: 首个针对多模态多智能体系统的层次化攻击框架，攻击成功率达 78.3%。
- **核心贡献**:
  - 将攻击分解为感知层（视觉/文本扰动）、通信层（拓扑操纵）、推理层（认知干扰）三层
  - 在 GQA benchmark 上评估 ReAct、Plan-and-Solve、Reflexion 三种范式
  - 发现推理层攻击最有效，超半数成功攻击导致多智能体一致出错
- **方法亮点**: HAM³ 通过联合扰动 vision-language 融合表示、腐化通信拓扑、偏置推理路径，构建完整攻击链；推理层注入最致命，因为错误会沿智能体间依赖传播。
- **为什么值得读**: MM-MAS 正快速落地生产环境，此框架首次系统量化其脆弱性，是设计鲁棒多模态 Agent 安全防线的必读基准。研究发现也直接指向防御优先级：通信拓扑保护比纯输入净化更关键。

## 2. Many-Shot CoT-ICL：让上下文学习真正"学会"推理
- **来源**: arXiv
- **作者**: Tsz Ting Chung et al.
- **链接**: http://arxiv.org/abs/2605.13511v1
- **方向**: 🤖 LLM
- **TL;DR**: 揭示多示例 CoT-ICL 的 scaling 规律与非推理任务截然不同，并提出 CDS 排序法带来最高 5.42pp 提升。
- **核心贡献**:
  - 发现推理任务中增加 CoT 示例对非推理型 LLM 不稳定，仅推理型 LLM 持续受益
  - 证明基于语义相似度的示例检索在推理任务上失效（过程兼容性 ≠ 语义相似性）
  - 提出 Curvilinear Demonstration Selection（CDS），利用概念渐进排序减少示例方差
- **方法亮点**: 把 many-shot CoT-ICL 重新框架为"上下文测试时学习"而非模式匹配，要求示例对目标模型易于理解、且排列形成平滑认知进程；CDS 在几何题 64 示例设置下提升 5.42pp。
- **为什么值得读**: 长上下文模型普及后 many-shot ICL 的使用量激增，但其 scaling 行为至今缺乏系统研究；本文填补了推理场景下的认知空白，对工程端提示工程和数据集构建均有直接指导意义。

## 3. Seg-Agent：无需训练的多模态链式推理语言引导分割
- **来源**: arXiv
- **作者**: Chao Hao et al.
- **链接**: http://arxiv.org/abs/2605.12953v1
- **方向**: 🔬 AI4Science（计算机视觉）
- **TL;DR**: 无训练框架通过视觉推理闭环实现与有训练方法媲美的语言引导分割性能。
- **核心贡献**:
  - 首个显式多模态链式推理（Explicit Multimodal CoT）的分割框架，完全无需参数更新
  - 利用 Set-of-Mark (SoM) 将候选区域直接渲染回图像，让 MLLM 在视觉域而非纯文本域推理
  - 引入 Various-LangSeg 基准，覆盖显式语义、通用目标、推理引导三类分割场景
- **方法亮点**: 生成-选择-精化三阶段视觉推理循环让 MLLM 迭代"看见"空间关系，克服了现有方法仅做文本抽象推理的盲区；整个流程对任意 MLLM + SAM 组合即插即用。
- **为什么值得读**: 在分割任务上消除训练需求意味着显著降低数据标注成本；同时 SoM 驱动的视觉反馈范式可迁移至机器人操作、医学影像等需要精细空间推理的场景。

## 4. OpenAaaS：面向分布式材料信息学的开源 Agent-as-a-Service 框架
- **来源**: arXiv
- **作者**: Peng Kang et al.
- **链接**: http://arxiv.org/abs/2605.13618v1
- **方向**: 🧪 自动化实验室 / AI4Science
- **TL;DR**: 提出"代码流动、数据不动"原则的分布式多智能体框架，实现跨机构材料数据安全协作。
- **核心贡献**:
  - Master Agent 仅做任务分解与规划，Sub-agents 在数据本地执行，原始数据永不离开源域
  - AlphaAgent 在深度分析问答中达 4.66/5.0，显著优于单次 RAG 基线
  - 超大规模六高熵合金描述符数据库服务验证了严格数据主权约束下的跨域集成
- **方法亮点**: 框架将 SaaS/IaaS 资源管理与 LLM Agent 推理能力结合，通过 hierarchical 架构天然支持机构间数据隐私；近数据执行模式避免了现有中心化平台的数据暴露风险。
- **为什么值得读**: 高熵合金、辐照钢等高性能材料研发依赖跨机构合作但数据高度敏感；OpenAaaS 提供了可立即复用的开源蓝图，对工业界和学术界都有落地价值。

## 5. Revisiting DAgger：为长时域 LLM Agent 找回经典算法
- **来源**: arXiv
- **作者**: Changhao Li et al.
- **链接**: http://arxiv.org/abs/2605.12913v1
- **方向**: 🤖 LLM / AI Agent
- **TL;DR**: 将经典 DAgger 迁移到多轮 LLM Agent 训练，4B/8B 模型在 SWE-bench 上分别达 27.3%/29.8%。
- **核心贡献**:
  - 识别 SFT（协变量偏移）与 RLVR（稀疏反馈）的互补缺陷，DAgger 同时解决两者
  - 基于学生-教师策略的 turn-level 插值采集轨迹，学生受到丰富的逐步监督
  - 4B DAgger agent 超越已发表的多个 8B SWE-agent；8B 超越 SWE-Gym-32B
- **方法亮点**: 在多轮交互中用插值比例控制探索-利用平衡，学生在真实环境状态分布下学习（不是离线的教师轨迹），从根本上消除训练-推理分布偏移；无需大规模 RL 基础设施。
- **为什么值得读**: 以参数量优势挑战更大模型证明训练策略比模型规模更关键；DAgger 的回归表明经典 imitation learning 理论在现代 LLM Agent 场景中仍有深刻洞察，值得再次重视。
